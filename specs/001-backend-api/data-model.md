# Data Model: Task Management API

## Entity: Task

### Fields
- **id**: Integer (Primary Key, Auto-increment)
  - Unique identifier for each task
  - Auto-generated by the database

- **title**: String (Required, Max length: 255)
  - The title or name of the task
  - Must not be empty

- **description**: String (Optional, Max length: 1000)
  - Detailed description of the task
  - Can be null or empty

- **completed**: Boolean (Required, Default: false)
  - Status of the task completion
  - True if completed, False if pending

- **created_at**: DateTime (Required, Auto-generated)
  - Timestamp when the task was created
  - Automatically set on creation

- **updated_at**: DateTime (Required, Auto-generated)
  - Timestamp when the task was last updated
  - Automatically updated on modification

### Validation Rules
- Title must be provided and not empty
- Title length must not exceed 255 characters
- Description length must not exceed 1000 characters
- Completed field defaults to False if not specified
- created_at and updated_at are automatically managed by the system

### Relationships
- No relationships with other entities in this initial implementation

### State Transitions
- A task can transition from completed=False to completed=True (and vice versa) via the toggle completion endpoint
- created_at is set once on creation and never changes
- updated_at is updated whenever any field changes

## Entity: TaskCreate (Input Model)
- **title**: String (Required)
- **description**: String (Optional)
- **completed**: Boolean (Optional, Default: false)

## Entity: TaskUpdate (Input Model)
- **title**: String (Optional)
- **description**: String (Optional)
- **completed**: Boolean (Optional)

## Entity: TaskResponse (Output Model)
- **id**: Integer
- **title**: String
- **description**: String
- **completed**: Boolean
- **created_at**: DateTime
- **updated_at**: DateTime
